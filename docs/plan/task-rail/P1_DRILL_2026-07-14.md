# P1 驗收 drill:兩 session 手動接力 2 步鏈(2026-07-14)

> §7 P1 驗收條款:「兩個互動 session 手動接力一條 2 步鏈;ledger 重播看得到完整交棒鏈。」
> 本檔為該 drill 的存證。結論:**通過**。

## 設定

- 乾淨 workspace(`edda init`,feat/task-rail-p1 的 debug binary)
- **Session A(producer)**:操作者主 session。建 2 步鏈 → 做實工(寫 `verbs.md`)→ `task done 1` 附收據與 evidence
- **Session B(verifier)**:獨立 agent session(全新 context,與 A 無共享記憶)。唯一協調管道是帳本:`task list --assignee verifier --status ready` 發現工作 → `show` 讀 brief → `start` 取租約 → 實讀 `verbs.md` 逐項核對六動詞 → `done` 附具體收據

沒有任何訊息傳遞、沒有輪詢 watcher——B 知道該做什麼,靠的只有帳本投影。

## 帳本重播(task.* 六事件,時序)

| seq | event | task | 內容 | 鏈 |
|---|---|---|---|---|
| 1 | `task.created` | #1 | step1: 產出 edda task 動詞清單檔 verbs.md(assignee producer) | ✓ |
| 2 | `task.created` | #2 | step2: 驗證 verbs.md 內容並簽收(assignee verifier,after #1,brief 載驗收條件) | ✓ |
| 3 | `task.started` | #1 | attempt 1,lease 3600s | ✓ |
| 4 | `task.done` | #1 | 收據「verbs.md 寫入完成,六動詞全列」,evidence `verbs.md` → **投影即時解鎖 #2 為 ready** | ✓ |
| 5 | `task.started` | #2 | attempt 1(B 取租約) | ✓ |
| 6 | `task.done` | #2 | 收據「實讀 drillws/verbs.md(共9行):第3-8行逐項核對,六個動詞全部列出,無缺漏無多餘項,判定通過」 | ✓ |

- 全帳本 hash chain 驗證:`total_events=7 FULL_CHAIN_VERIFIED=true`(含 init 事件;task.* 入全域鏈,不另起鏈)
- 終態:`#1 [done]`、`#2 [done]`,各 attempts: 1
